Understands scoping in JavaScript and knows to always use const if possible, otherwise let.

Scope defines where variables can be accessed in your code.

  • Global scope: Variables declared outside any function or block. Accessible from anywhere in the code.
  • Function scope: Variables declared inside a function. Accessible only within that function.
  • Block scope: Variables declared with let or const inside a block { }. Accessible only inside that block.

Use const by default to declare variables that should not be reassigned.

Use let when you need to reassign a value later.

Don't use var because it ignores block scope and can cause unexpected behavior.

Understands advanced concepts about functions in JavaScript.

Functions in JavaScript are first-class citizens, meaning they can be:

  • Stored in variables
  • Passed as arguments to other functions
  • Returned from other functions

Can write callback functions with different syntax.


Regular function
setTimeout(function() {
  console.log("Hello");
}, 1000);

Arrow function
setTimeout(() => console.log("Hello"), 1000);

Callback function
function greet() {
  console.log("Hello");
}
setTimeout(greet, 1000);

Understands the function of NPM and can compare it with other package managers.

NPM (Node Package Manager) is a tool that helps manage JavaScript project dependencies.

  • Installs third-party packages from the NPM registry.
  • Manages project dependencies and their versions.
  • Runs scripts defined in the package.json file (e.g. npm start, npm test).

Other package managers:

  • Yarn: Faster installs, improved caching, and better dependency locking.
  • pnpm: Uses hard links to save disk space and ensures efficient dependency management.

Understands the different purposes of package.json and can set it up by hand.

The package.json file is the heart of a Node.js project. It defines project information and configuration.

  • Metadata: name, version, author, and license.
  • Dependencies: lists the packages your project needs.
  • Scripts: defines custom commands to run with npm run.
  • Entry point: specifies the main file (e.g. index.js).

You can create it manually or generate it using npm init.

Can create a server with Express from memory.


const express = require('express');
const app = express();

const PORT = 8080;
app.listen(PORT, (error) => {
    if (error) {
        console.log("Error starting the server", error);
        return;
    }
    console.log("Server is running on port", PORT);
});

Understands node_modules.

node_modules is a folder where all installed dependencies are stored.

It is created automatically when you run npm install.

Each dependency can have its own dependencies, forming a dependency tree.

Never edit files inside node_modules directly.

Usually, this folder is ignored in .gitignore because it can be recreated from package.json.
