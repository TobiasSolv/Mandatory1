Understands the use case for Nodemon and its limitation for development.

Nodemon is a development tool that automatically restarts your Node.js server whenever files change.

  • It helps speed up development by removing the need to restart manually.
  • It watches files in your project and reloads the server on save.
  • It should only be used in development, not in production.
  • Limitations: it does not handle code crashes or production-level process management.

Can use loop methods (map, filter, reduce) and choose the right one at the appropriate time.

  • map(): transforms each element and returns a new array.
  • filter(): keeps elements that match a condition.
  • reduce(): combines all elements into a single value.

Understands why a functional approach to loops is preferred and how to implement it without side effects.

A functional approach avoids changing external variables and makes code easier to read and maintain.

  • Each function (like map or filter) returns a new array instead of modifying the original one.
  • This makes data flow predictable and helps prevent bugs caused by shared state.

Can use the right terminology when it comes to the different parts of a URL.

A URL (Uniform Resource Locator) identifies the location of a resource on the internet.


https://example.com:8080/users?id=27#profile
  • Protocol: https://: defines how data is transferred.
  • Domain: example.com: identifies the server.
  • Port: :8080: specifies the connection port.
  • Path: /users: specifies the resource location.
  • Query string: ?id=27: sends data as key/value pairs.
  • Fragment: #profile: points to a specific section on the page.
