Understand and work with data types in JavaScript.

JavaScript has several built-in data types used to store and work with different kinds of values.

  • string: text values, "Hello"
  • number: numeric values, 42 or 3.14
  • boolean: true or false
  • null: empty value
  • undefined: variable declared but not assigned a value
  • object: collection of key/value pairs
  • array: ordered list of values
  • function: reusable block of code
  • date: represents dates and times

Understands type coercion and strategies to avoid it.

Type coercion is when JavaScript automatically converts one data type to another, for example a number to a string.

It can happen in operations like '5' + 1 which becomes '51', or '5' - 1 which becomes 4.

To avoid unexpected results, use this equality === instead of ==.

Make sure to convert types manually using Number(), String(), or Boolean().

Can design a REST API.

A REST API uses HTTP methods to perform actions on resources such as users or products.

  • GET /users: list all users
  • POST /users: create a new user
  • GET /users/:id: get one specific user
  • PUT /users/:id: update an existing user
  • DELETE /users/:id: delete a user

Knows the three REST conventions we follow:

  1. Using the right HTTP verb

    HTTP request methods (verbs): GET, POST, PUT, PATCH, DELETE

  2. Ordering

    Keep a consistent URL structure, for example: /collection/id/subcollection/id

  3. Naming

    Use nouns and plural names for collections, e.g. /users, /orders
