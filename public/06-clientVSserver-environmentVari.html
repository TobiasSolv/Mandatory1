Can define and run scripts with NPM.

NPM scripts are custom commands defined in the package.json file to automate tasks such as starting a server, building, or testing code.


    start: node index.js
    dev: nodemon index.js
    build: echo Building project...

To run scripts:


npm run dev
npm run start

Knows how to define environment variables natively in Node.js (without any libraries).

Environment variables are used to store sensitive or configurable data outside the code (port numbers, API keys).

  • Define them in your terminal before running your app:

# macOS / Linux
PORT=8080 node app.js

# Windows (PowerShell)
$env:PORT=8080; node app.js
  • Access them in Node.js using process.env:

const PORT = process.env.PORT || 8080;
app.listen(PORT, (error) => {
    if (error) {
        console.log("Error starting the server", error);
        return;
    }
    console.log("Server is running on port", PORT);
});

Fetching in the client vs. fetching in the backend.

Client-side fetching happens in the browser, usually with fetch() in frontend JavaScript.

  • Runs after the page loads.
  • Used to display or update data dynamically.
  • Can be seen in the browserâ€™s network tab (less secure).

// Client-side fetch
fetch("/api/users")
  .then(res => res.json())
  .then(data => console.log(data));

Backend fetching happens in a Node.js server, usually with fetch or axios.

  • Runs on the server before sending a response to the client.
  • Useful for communicating with external APIs securely.

// Backend fetch (Node.js)
const response = await fetch("https://api.example.com/data");
const data = await response.json();
console.log(data);

Structuring HTML with semantic HTML.

Semantic HTML uses elements to describe the content structure, improving accessibility and SEO.

  • <header>: top section of the page (e.g. logo, navigation)
  • <nav>: main navigation links
  • <main>: main page content
  • <section>: grouped related content
  • <article>: independent piece of content
  • <aside>: side content like ads or related links
  • <footer>: bottom section (e.g. copyright, contact info)

Can point out which files are client files and which are server files.

Client files:

  • .html, .css, .js
  • Run in the browser and handle user interaction, layout, and visuals.

Server files:

  • .js files that run in Node.js.
  • Handle API routes, logic, database queries, and server responses.

Can argue for a better way to structure the frontend (assets, pages).

A well-structured frontend keeps files organized and easy to maintain.

A recommended structure is:


/public
  /assets
    /images
    /css
    /js
  /pages
    index.html
    about.html
  • assets/: reusable static files like images, CSS, and JS.
  • pages/: separate HTML pages or views for different parts of the site.
  • Helps keep content, styling, and logic separated and easier to manage.
