Understands how code and its placement affects application memory and efficiency.

Code structure and placement can greatly impact how efficiently an application uses memory and CPU resources.

  • Functions and variables should be declared only when needed to reduce memory usage.
  • Use local scope to limit memory allocation and prevent global memory leaks.
  • Avoid unnecessary imports or large libraries that increase bundle size.
  • Use lazy loading or dynamic imports to load code only when it’s needed.
  • Optimize loops and data structures to reduce unnecessary processing.

Efficient code placement leads to faster load times and lower memory consumption, especially in large or long-running applications.

Understands the arguments for and against SSR vs. CSR when it comes to:

  1. Load time: SSR (Server-Side Rendering) sends pre-rendered HTML to the client, so content appears faster. CSR (Client-Side Rendering) loads a blank page first and renders content with JavaScript, so the first load is slower but navigation is faster after initial load.
  2. Resources spent (duration, when and where): SSR uses server resources to render each page request. CSR uses the client’s (browser’s) resources, reducing server load but increasing client work.
  3. SEO: SSR is better for SEO because search engines can immediately read the rendered HTML. CSR can have SEO issues unless pre-rendering or hydration is used.
  4. Dealing with CORS: SSR avoids CORS issues since all API requests can be made directly from the server. CSR must follow CORS rules when fetching from external APIs.

Can argue for better ways to structure the code and how to avoid files that are too long.

Well-structured code improves readability, maintainability, and scalability.

  • Split long files into smaller modules or components.
  • Follow clear folder structures (/routes, /controllers, /utils).
  • Group related logic together and separate concerns (business logic vs. UI logic).
  • Use helper functions or utility modules for repetitive logic.
  • Keep functions small and focused — each should do one thing well.
  • Adopt consistent naming conventions and code formatting (use ESLint and Prettier).
